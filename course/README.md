# Учебный курс по Python

Каталог содержит исходники курса «Основы Python: практика с CodeRunner». Курс хранится в Git как обычные PHP-массивы, поэтому условия, теорию и тесты можно проверить до установки.

## Состав курса

В курсе восемь тем:

1. ввод, переменные и арифметика;
2. условия;
3. циклы;
4. списки и строки;
5. простые алгоритмы;
6. функции и вещественные числа;
7. классы;
8. `asyncio`, потоки и процессы.

В каждой теме есть страница с теорией и тест из трёх задач CodeRunner. Всего 9 страниц и 24 задачи. В каждой задаче два открытых примера и один скрытый тест. Обычный текстовый редактор выбран специально: он стабильно работает даже при холодном кэше браузера.

Первые 15 заданий проверяют программы со стандартным вводом. Задания 16–18 вызывают функции и сравнивают `float` с погрешностью. Задания 19–21 создают объекты. Задания 22–24 проверяют `asyncio.gather`, `ThreadPoolExecutor` и `ProcessPoolExecutor`. Число рабочих потоков и процессов ограничено двумя, поэтому эталоны работают на стенде с 4 ГБ памяти.

## Установка в лабораторном стенде

Ничего импортировать вручную не нужно. `moodle/entrypoint.sh` запускает установщик после установки Moodle и плагинов. Повторный запуск находит курс по shortname `PYTHON-CR-START`, добавляет отсутствующие материалы и не создаёт копию. Пользователи и старые попытки сохраняются.

Проверка структуры и всех эталонных решений:

```bash
docker compose exec -T moodle php /opt/python-course/check.php --run-reference
```

Каждое эталонное решение при этой проверке реально выполняется через CodeRunner и Jobe.

## Установка на другом Moodle 5.2.1

Сначала установите и проверьте CodeRunner. Затем на сервере из корня репозитория выполните:

```bash
sudo -u www-data env MOODLE_CONFIG=/var/www/moodle/config.php \
  php course/install.php
```

Замените путь к `config.php`, если Moodle установлен в другом каталоге. По умолчанию курс создаётся в категории с ID `1`. Другую категорию можно указать так:

```bash
sudo -u www-data env \
  MOODLE_CONFIG=/var/www/moodle/config.php \
  PYTHON_COURSE_CATEGORY_ID=3 \
  php course/install.php
```

После установки:

1. откройте «Администрирование сайта → Курсы → Управление курсами»;
2. найдите `PYTHON-CR-START`;
3. запишите преподавателей и студентов стандартными средствами Moodle;
4. откройте любую практику и отправьте правильное и ошибочное решение;
5. проверьте кнопку ИИ с ролью студента.

Повторный запуск безопасно дополняет созданный этим проектом курс: синхронизирует его страницы, добавляет отсутствующие разделы, тесты и вопросы. Существующие вопросы, пользователи и попытки не удаляются. Если преподаватель вручную менял страницы курса, сначала проверьте обновление на копии: текст этих страниц будет заменён версией из `content.php`.

## Обновление лабораторного сервера

Из каталога репозитория:

```bash
git pull
docker compose up -d --build
./scripts/check.sh
./scripts/smoke-test.sh
```

После запуска Moodle сам дополнит существующий `PYTHON-CR-START`. Отдельный импорт и удаление старого курса не нужны.

## Файлы

- `content.php` — теория, условия, эталонные решения и тесты;
- `install.php` — создание разделов, страниц, банка вопросов и тестов через Moodle API;
- `check.php` — проверка структуры, скрытых тестов и запуск эталонных решений через Jobe.

Исходники Moodle и CodeRunner курс не изменяет.
