# Moodle AI CodeRunner Lab

## Что это

Учебный стенд на Moodle 5.2.1 для проверки программ и получения подсказок по решению:

- Moodle хранит курс, вопросы и попытки;
- CodeRunner управляет тестами;
- Jobe выполняет Python в отдельном контейнере;
- AI service разбирает код и результат проверки;
- Ollama запускает локальную модель;
- `local_aicodehelper` показывает анализ внутри попытки CodeRunner.

Наружу опубликован только Moodle. Пользовательский код не запускается в Moodle или AI service.

## Как всё связано

```text
Студент
   ↓
Moodle → MariaDB
   ↓
CodeRunner
   ↓
Jobe

После проверки:
Moodle → local_aicodehelper → AI service → Ollama
```

## Что было готовым

- Moodle 5.2.1;
- CodeRunner и обязательный behaviour-плагин;
- официальный Jobe Server;
- Ollama;
- небольшая модель Qwen.

## Что разработано в проекте

- Docker-развёртывание всего стенда;
- FastAPI AI service;
- безопасный статический анализ Python через `ast`;
- fallback без Ollama;
- Moodle-плагин `local_aicodehelper`;
- кнопка анализа внутри попытки CodeRunner;
- фильтрация скрытых тестов и AI-кэш;
- автоматические тесты и инструкции.

Версии Moodle и сторонних компонентов закреплены в `moodle/Dockerfile` и `compose.yaml`.

## Быстрый запуск на Mac

Нужен Docker Desktop с Compose. Рекомендуется 6 ГБ RAM и 20 ГБ свободного места.

```bash
make setup
make build
make up
make check
make smoke
```

`make setup` создаёт локальный `.env` и генерирует пароли. При первом старте Jobe собирается, а Ollama скачивает модель примерно на 400 МБ. Ход загрузки виден так:

```bash
docker compose logs -f ollama-model
```

Moodle по умолчанию откроется на [http://localhost:8080](http://localhost:8080).

Администратор создаётся только при первой установке. Логин и пароль находятся в `.env`:

```bash
grep '^MOODLE_ADMIN_' .env
```

Модель выбирается там же:

```dotenv
OLLAMA_MODEL=qwen2.5-coder:0.5b
```

После смены `OLLAMA_MODEL` выполните `make up`. Загрузчик проверит volume Ollama и скачает модель только при её отсутствии.

## Быстрый запуск на Ubuntu

После установки Docker Engine, Docker Compose Plugin и Git весь стенд можно клонировать и запустить одной командой. Замените IP в команде на адрес сервера:

```bash
git clone https://github.com/Victor-git-19/moodle-ai-coderunner-lab.git && cd moodle-ai-coderunner-lab && MOODLE_URL=http://192.168.1.50:8080 ./scripts/deploy-server.sh
```

Скрипт сам создаст `.env`, сгенерирует пароли, соберёт контейнеры, дождётся сервисов и выполнит проверки. Если порт `8080` занят, сразу выберите другой:

```bash
git clone https://github.com/Victor-git-19/moodle-ai-coderunner-lab.git && cd moodle-ai-coderunner-lab && MOODLE_PORT=8081 MOODLE_URL=http://192.168.1.50:8081 ./scripts/deploy-server.sh
```

Jobe проекта работает только во внутренней Docker-сети и не публикует порт. Поэтому уже работающий на сервере Jobe с портом `9000` не мешает стенду. Полная инструкция: [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).

## Проверка работы

Сначала выполните:

```bash
make check
make smoke
```

`check` проверяет состояние пяти сервисов. `smoke` реально запускает Python через Jobe, вызывает AI service, проверяет fallback, плагины, sesskey и отсутствие скрытых данных в AI payload.

### CodeRunner вручную

1. Войдите администратором и создайте курс.
2. Добавьте элемент «Тест» и вопрос типа `CodeRunner`.
3. Выберите тип вопроса `python3`.
4. Укажите ответ `print("Hello from Jobe")`.
5. Добавьте тест без входных данных с ожидаемым выводом `Hello from Jobe`.
6. Откройте предпросмотр, отправьте ответ и нажмите «Проверить».

Адрес Jobe находится в «Администрирование сайта → Плагины → Типы вопросов → CodeRunner». В Compose его значение — `jobe`, без `http://`.

### ИИ-анализ в попытке

После проверки CodeRunner появляется кнопка «Проанализировать решение с помощью ИИ». Moodle сам отправляет условие, код, оценку и разрешённые результаты тестов. Результат показывается под CodeRunner. Повторное нажатие возвращает результат из серверного кэша.

Настройки находятся в «Администрирование сайта → Плагины → Локальные плагины → ИИ-помощник по коду». Там можно включить интеграцию, выбрать режим подсказок, лимит запросов и timeout. Полный исправленный код по умолчанию запрещён. Пользователю нужна capability `local/aicodehelper:analyzeattempt`.

Страница `/local/aicodehelper/index.php` — только диагностика AI service и модели без создания попытки CodeRunner. Основной сценарий студента — кнопка в попытке.

### Как настроить скрытый тест

В вопросе CodeRunner для скрытого теста:

- не включайте «Use as example»;
- в поле `Display` не выбирайте `SHOW`; используйте скрытый режим, например `HIDE_IF_FAIL`;
- проверьте вопрос от роли студента: вход, ожидаемый результат и test code не должны быть видны.

Плагин отправляет для такого теста только номер, признак успеха, безопасный тип ошибки и короткое сообщение. Вход, ожидаемый ответ, test code, данные шаблона и эталонное решение не передаются.

Демонстрация:

1. Студент отправляет ошибочное решение.
2. CodeRunner показывает провал теста.
3. Студент нажимает кнопку ИИ.
4. ИИ объясняет вероятную причину, крайние случаи и следующий шаг.
5. Студент исправляет решение и повторяет проверку.

Логи отдельного сервиса:

```bash
docker compose logs -f moodle
docker compose logs -f ai-service ollama
```

## Где хранятся данные

- `db_data` — база Moodle;
- `moodledata` — файлы Moodle и `config.php`;
- `ollama_data` — скачанные модели.

`make down`, `make up` и `make restart` сохраняют эти volumes. Полное удаление требует явного подтверждения:

```bash
make reset CONFIRM=yes
```

Команда безвозвратно удаляет базу, файлы Moodle и модели.

## Безопасность

- код выполняется только в Jobe;
- AI service разбирает код, но не запускает его;
- скрытые тесты фильтруются строгим allowlist;
- AJAX требует авторизацию, capability и sesskey;
- запрос к AI service выполняет сервер Moodle;
- наружу открыт только Moodle.

Фиксированная внутренняя подсеть Compose сохранена намеренно: без неё защищённый HTTP-клиент Moodle 5.2.1 блокирует обращение CodeRunner к Jobe как к приватному адресу.

## Ограничения

- это учебный стенд для локальной сети, без HTTPS;
- небольшая модель может ошибаться;
- AI даёт подсказку и не заменяет результат CodeRunner;
- перенос на действующий Moodle нужно сначала проверять на копии.
